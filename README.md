## About Me

I'm an Informatics Engineering student who learns by **building real products**, not following tutorials. My journey started with curiosity about how systems work it evolved into building solutions that solve actual problems.

Most of my energy goes into **CraftSMS**, a modern School Management SaaS I'm building to help educational institutions digitize operations. I believe the best way to learn software engineering is by shipping products that create impact.

I'm deeply interested in:
- **Full Stack Development** — Building end-to-end systems
- **SaaS Architecture** — Multi-tenant, scalable platforms
- **Cloud Computing** — Modern infrastructure patterns
- **Agentic Engineering** — Autonomous AI systems
- **AI-Assisted Development** — Leveraging LLMs for better code

When I'm not coding, I'm reading about system design, architecture patterns, and the latest in AI/ML.

## 🏫 Current Mission: Building CraftSMS

<div style="background: linear-gradient(135deg, rgba(59, 130, 246, 0.1), rgba(37, 99, 235, 0.1)); border-left: 3px solid #3b82f6; padding: 24px; border-radius: 12px; margin: 24px 0;">

**CraftSMS** is a modern multi-tenant School Management SaaS designed to simplify educational administration and improve digital adoption in schools.

**What it does:**
- Student Management & Records
- Attendance & Engagement Tracking
- Academic Progress Monitoring
- School Communication Platform
- Finance & Payment Management
- Staff & Resource Management

**Architecture:**
- Modern, responsive Next.js frontend
- Python FastAPI backend
- PostgreSQL + Supabase for data
- Docker containerization
- GitHub Actions CI/CD

**Links:**
- 🌐 [Live Demo](https://craftsms.com)
- 💻 [Repository](https://github.com/swaraylahais61-cloud/CRAFT-SMS)
- 📖 [Documentation](https://craftsms.com/docs)
